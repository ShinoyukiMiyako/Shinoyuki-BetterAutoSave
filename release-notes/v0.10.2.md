## v0.10.2 — 存档安全加固

围绕异步保存的失败路径与并发边界做系统性加固，经六轮交叉审查与对抗核查，修复 7 Critical / 10 Major / 17 Minor，单元测试 73 -> 139。功能与配置不变

### 数据安全

- ChunkMapSaveMixin 派发异常后还原 isUnsaved 并复位状态机，vanilla 同步保存当场兜底，不再卡死 SERIALIZING
- chunk / entity IO 失败改为已序列化 tag 原地重投，不依赖对象仍加载；重试耗尽走坐标恢复队列或 ERROR 明示维度坐标
- 在途保存与新编辑碰撞时主线程补拍最新代入接力槽，旧 IO 落地后由 worker 重投——"编辑后立即卸载"不再丢最新一代（相对 vanilla 的净回归修复，chunk / entity 对称）
- 关服收尾显式 drain 恢复队列；degraded 模式下 tick drain 不再停摆
- autosave 通道对在途 chunk 不再静默吞为 fallback
- 接力链落盘的最新代 tag 补派发 ChunkDataEvent.Save，依赖该事件向 tag 写增量的第三方 mod 不再丢接力代增量
- worker 未捕获异常 (onUnhandledError) 改为接力槽前置消费，不再丢弃碰撞最新代；chunk / entity 对称
- in-flight 碰撞分支补 vanilla 副作用 POI flush (chunk) 与 emptyChunks remove (entity)，对齐常规路径

### 并发与资源

- mustDrain / serializing / 队列深度 gauge 改 CAS 结果驱动配平，消除终态竞态假挂
- SavedData：大文件同步 fallback 加 try-finally 释放在途占位、原子写（tmp + fsync + rename）、dirty 标志 volatile 镜像收口跨线程可见性、同名文件在途去重、offer 失败与异常同路补偿
- SavedData 乐观清 dirty 上移到入队之前，修 worker 失败 re-mark 被主线程覆盖的 lost-update，下周期正确重试
- EntitySaveState 状态表 CLEAN_LANDED 终态安全剔除，消除长跑无界增长
- 删除 entity 侧从未被消费的调度链死代码

### 观测

- SavedData 队列深度纳入 Prometheus 与周期日志
- hottest-chunks 统计 key 纳入维度，跨维度同坐标不再串样
- 关服日志区分"worker 线程终止"与"IO 已落盘"两个判定

### 升级

替换 jar 即可，配置不变
