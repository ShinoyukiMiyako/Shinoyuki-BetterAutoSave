## v0.11.0 — 接力协议重构

pendingSnapshot 接力协议（在途 IO 与新编辑碰撞时把最新代快照接力落盘的机制）的协议字段原是四个独立原子单元，没有单一线性化点，多轮交错攻击证实补丁无法收敛。本版将其收口为单字 SlotWord CAS 状态机并对碰撞、终态、关服路径系统性封口，修复 10 Critical / 3 Major / 1 Minor，单元测试 139 -> 179。功能与配置不变，每条按"调用点：0.10 后果；现在行为"对照

### 接力协议（chunk）

- ChunkMapSaveMixin 碰撞分支旧序 capture -> dispatch -> register：0.10 在飞 IO 恰于 dispatch 期间落地时取到空槽，之后才登记的接力快照成无人消费的孤儿，最新代静默丢失；现在登记前置于派发，回调任何时刻可见接力槽
- 同分支 register 前置后，dispatch 期间第三方 ChunkDataEvent.Save listener 仍在主线程改写 tag：0.10 在飞回调会把未就绪 tag 取走重投，三线程无栅栏共享同一 NBT，落盘可能损坏；接力槽升级 EMPTY/PREPARING/READY 三态，dispatch 返回前消费权不外放，回调只标 missed 由主线程 publish 自踢
- ChunkSaveTask 终态死亡出口（onUnhandledError / 重试耗尽）旧用 drainPendingSnapshot 无条件夺槽：0.10 同样会夺走主线程改写中的 PREPARING tag；改 takeReadyForTerminalConsumer 原子三态消费，READY 直接接力、PREPARING 交还主线程、EMPTY 才走 vanilla 兜底
- enterSerializing 开新周期时跨周期残留的 missed 标记：0.10 新周期误继承后，主线程在本代 IO 未落地时并发重投，多 worker 下旧代覆盖新代静默丢数据；改代际序号 fence（cycleSeq 全局单调），missed 只在同周期有效
- 落地与取槽拆两步（ioCompletedSuccessfully + takeReadyPendingSnapshot）：0.10 两步之间被切走会把 missed 标进错误周期，引发提前接力或误判 CLEAN 提前清账；landAndTake 合并为单 CAS，missed 恒以落地时刻周期标定
- 协议字段（phase / mustDrain / 接力槽 / 在飞代）收口单字 SlotWord：0.10 终态消费者清 mustDrain 与主线程 begin 拉槽交错时 gauge 掉 0，关服 join 漏等在途接力丢增量；单 CAS 后槽非空必然 gauge >= 1
- begin 落在回调已 CLEAN_LANDED 退出之后（phase==CLEAN）：0.10 两个自踢触发点均不命中，publish 发布永无消费者的 READY 孤儿，卸载路径增量静默丢失且 mustDrainPending 永久 +1；begin 同 CAS 立无在飞消费者标记，publish 据此把 pending 交还主线程当场重投
- 终态失败 HANDED_TO_MAIN 分支的 markMustDrain 覆盖 TERMINAL_HANDED：0.10 一个 publish 自踢触发点失活（冗余触发点兜底，不丢数据）；移除覆盖，双触发点独立可用

### 实体路径

- EntitySaveTask CLEAN_LANDED 分支不取槽直接 evict：0.10 回调 stale 读旧代误判 CLEAN 时，主线程刚登记的最新实体列表随状态对象送 GC，实体已被 vanilla 驱逐无任何恢复通道，永久静默丢失；现在 evict 前必取槽，非空走接力重投
- EntityStorageMixin 登记与回调取槽互不可见：0.10 回调 REQUEUE 取槽早于登记时取到 null 跳过重投，pending 落进死状态永久滞留且 mustDrainPending 永久 +1；建立写后读对偶（回调先写 phase 终态再取槽，主线程先写槽再重读 phase），getAndSet 裁定唯一消费者，主线程检测到无在飞消费者即自踢重投
- EntitySaveState.ioFailed REQUEUE 原地重投瞬态发布 DIRTY：0.10 主线程重读恰中该窗口会误判无在飞消费者偷走接力槽，与重投回调双在飞并覆盖 inFlightGeneration；重投全程保持 IO_PENDING，DIRTY 收窄为"无消费者等下周期"的真终态
- entity 碰撞分支 capturePending 抛（多为 OOM）后不清 mustDrain：0.10 该坐标 gauge 永久 +1；catch 内补 CAS 清理，与 chunk 撤销路径对称

### IO 失败与关服收尾

- submitIo 内 submit 同步抛被吞进无人 join 的废 future：0.10 task 永卡 IO_PENDING，inFlightIoPending 永久 +1，关服 drainPending 永不见零；submit 自包 try，同步抛配平 gauge 后走 IO 失败既有终态补偿
- REQUEUE 接力重投点 sink 同步抛（OOM）：0.10 已取走的接力代彻底丢失，serializing gauge 泄漏毒化关服判定；safeReoffer 包裹重投，sink 自包 inc/dec 配平，回调链尾 exceptionally 兜底，任何路径不静默
- workersStopping 残窗（关服后接力迟到）分支只 log 不清账：0.10 mustDrain 成无主孤儿，诊断空闲门失效、drain-unload 误报超时、Prometheus 指标长期失真；残窗分支真正 CAS 清理，幂等防双 dec
